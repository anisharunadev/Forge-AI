# Forge AI Documentation Site

The official documentation for **Forge AI — the Enterprise AI SDLC Operating System**, built with [Astro](https://astro.build/) + [Starlight](https://starlight.astro.build/).

> **Source SHA:** `forareal-final` · **Generated by:** DocAgent ([FORA-298](/FORA/issues/FORA-298)) · **Freshness:** 2026-06-18

## What's inside

This site is the single source of truth for the Forge AI platform surface:

- **Overview** — what Forge AI is, the quickstart, and the headline features.
- **Installation** — prerequisites, dev setup, and production deploy.
- **Self-hosting** — AWS reference architecture, Kubernetes (EKS), environment variables.
- **Agents** — every sub-agent: Master Orchestrator, BA, Architect, Developer, QA, Security, DevOps, Documentation, Memory.
- **Architecture** — staged workflow, Knowledge Layer, multi-tenancy, audit log.
- **Integrations** — every priority-1 MCP server (Jira, GitHub, Confluence, SonarQube, Figma, AWS, Slack/Teams).
- **API reference** — OpenAPI 3.1 contract for the public surface.
- **Security** — threat model, IAM, secrets, compliance posture.
- **Reference** — glossary, ADR index.

All content is generated from the Forge AI **Knowledge Layer** (`workspace/`) — the doc index is the **single source of truth**, and this site is the read-mostly surface that ships from it.

## How content stays fresh

This site is wired to the [Documentation Agent](/FORA/issues/FORA-298) pipeline:

1. **`workspace/project/PRD.md`**, **`roadmap.md`**, **`tech-stack.md`** are the source of truth for product facts.
2. **`workspace/memory/architecture.md`** is the source of truth for design facts.
3. **`workspace/customer/{glossary,standards,conventions}.md`** are the source of truth for customer-facing vocabulary.
4. Every page in `src/content/docs/` carries a `source_sha`, a `last_generated_at`, and a freshness footer.
5. On every doc-generation run, the DocAgent rebuilds pages from the Knowledge Layer and emits a doc-index entry in `workspace/project/docs.md` with a new `content_sha`.

A re-run with the same inputs is a **byte-identical no-op** (idempotency contract).
