The **Documentation agent** is the seventh and final stage. It wakes when a deploy is verified and produces the **Confluence page**, the **ADR links**, and the **final audit row** that closes the run.

## What it reads

- The PRD, ADR, PR, QA report, Security report, and release notes from earlier stages.
- The customer's conventions + glossary.
- The Knowledge Layer's `workspace/project/docs.md` (the doc index).

## What it produces

| Artefact | Storage |
| --- | --- |
| Confluence page | Confluence (via the Confluence MCP server) |
| ADR links | The Confluence page embeds them |
| Doc index entry | `workspace/project/docs.md` (the `DocIndexEntry` row) |
| Audit row | Audit-account S3 (the run-closure row) |
| Changelog entry | `CHANGELOG.md` |
| Release notes | `docs/release-notes/RELEASE_NOTES_<v>.md` |

## The doc index

Every doc run appends or refreshes one entry per artefact in [`workspace/project/docs.md`](https://github.com/fora-platform/fora/blob/main/workspace/project/docs.md):

```typescript
{
  path: 'docs/api/openapi.yaml',
  kind: 'api_docs',
  title: 'Forge AI Public API (OpenAPI 3.1)',
  last_generated_at: '2026-06-18T00:00:00Z',
  source_commit: 'forareal-final',
  generator: 'api_docs',
  version: '1.0',
  content_sha: 'sha256:...',
  approval_required: false,
  tags: ['api', 'openapi'],
  architecture_area: 'runtime',
}
```

The DocIndex entries are the **storage contract** for the four generators (`readme`, `api_docs`, `changelog`, `release_notes`, `adr`).

## The freshness SLA

| Kind | Cadence | Max age | Blocks release? |
| --- | --- | --- | --- |
| `readme` | every release | 7 d | warn-only |
| `changelog` | every release | 7 d | warn-only |
| `release_notes` | every release | 30 d | warn-only |
| `api_docs` | every merge | 24 h | **blocks** |
| `adr` | once (per decision) | 365 d | warn-only |

The Memory Agent reads this index; the Audit Agent audits writes to it; the Master Orchestrator consults `docs.freshness_check()` before the release stage.

## The idempotency contract

Same `input_sha` + `source_commit` + `content_sha` → byte-identical `DocIndexEntry`. A re-run is a no-op unless one of the three inputs changed.

## The audit-row

The run-closure audit row looks like:

```json
{
  "id": "01HXYZ...",
  "tenant_id": "acme-corp",
  "run_id": "01HXYZ...",
  "stage": "docs",
  "tool": "doc.run_close",
  "actor": "agent:documentation",
  "input_sha": "sha256:...",
  "output_sha": "sha256:...",
  "args_hash": "sha256:...",
  "started_at": "2026-06-18T00:00:00Z",
  "ended_at": "2026-06-18T00:00:01Z",
  "tokens_in": 1234,
  "tokens_out": 567,
  "usd": 0.04,
  "result": "ok",
  "metadata": {
    "confluence_page_id": "12345",
    "adr_ids": ["0001", "0042"],
    "doc_index_entry_ids": ["openapi.yaml"]
  }
}
```

The run is closed when this row ships to the audit account.
