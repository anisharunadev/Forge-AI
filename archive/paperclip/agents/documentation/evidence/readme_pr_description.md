### What this PR does

Adds the first `README.md` to the Forge AI monorepo, generated from project memory + customer conventions per Forge AI-120 ¬ß"Scope":

- `README.md` ‚Äî the new entry-point doc. Sections rendered from
  `workspace/project/PRD.md`, `workspace/project/roadmap.md`,
  `workspace/project/tech-stack.md`, `workspace/customer/standards.md`,
  and `workspace/customer/conventions.md`.
- `workspace/project/docs.md` ‚Äî refreshed doc index: the README entry
  is added/updated; existing CHANGELOG/API/ADR entries are preserved.
- `agents/documentation/readme_generator.py` ‚Äî the generator
  implementation (the deliverable).
- `agents/documentation/readme_smoke_test.py` ‚Äî the smoke test
  (Forge AI-120 acceptance battery: 47 assertions, all PASS).

### Approval routing

`approval_required=True` because this is the first generation (no
`README.md` existed before). Per the contract in `prompt.md` ¬ß"Hard
constraints" item 3:

- **First generation** ‚Üí `approval_required=True` (README rewrite).
- **Subsequent run with identical content** ‚Üí `approval_required=False`
  (no-op, auto-merge).
- **Subsequent run with changed content** ‚Üí `approval_required=True`
  (rewrite).

### Idempotency contract

Re-running with the same `input_sha` produces byte-identical body.
The README's freshness stamp lives on the `DocArtifact` wrapper, not
in the body ‚Äî so wall-clock changes do not break re-run determinism.

- `content_sha` of the artifact: `1342571d631a488bfb44018bfc3a3bef440cb2695de8345ad37f1d3a90450848`
- `source_sha` of the artifact: `forareal-final`
- `freshness_timestamp`: stamped at run time on the artifact, not
  embedded in the body.

### Cost record

- `model`: `claude-sonnet-4-6` (default per `prompt.md`; not invoked
  in v1 because the renderer is deterministic and structure-only)
- `tokens_in`: `0`
- `tokens_out`: `0`
- `usd`: `0.0`
- `fallback_used`: `false`

The structure-only renderer satisfies the determinism + idempotency
contract more cheaply than a model call. An LLM-based renderer can
be layered later (for prose polish) without changing the
`DocArtifact` contract.

### Risk and rollback

- **Risk:** the generated README is a new entry-point file. If a
  customer-facing diff review surfaces an inaccurate statement
  (e.g., a stale license line), revert this PR and re-run the
  generator with the corrected memory.
- **Rollback:** the generator is pure-functional; deleting
  `README.md` and reverting `workspace/project/docs.md` restores the
  pre-PR state byte-for-byte. The `readme_generator.py` and
  `readme_smoke_test.py` files can be kept in tree as future
  references.
