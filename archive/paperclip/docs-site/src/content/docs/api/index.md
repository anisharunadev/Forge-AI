The **Forge AI public API** is the customer-facing surface. REST over HTTPS, JSON payloads, OpenAPI 3.1 published from the same source of truth as the validator. Generated by the [API docs generator](/agents/documentation/).

## Design principles (per [`memory/architecture.md` §7](https://github.com/fora-platform/fora/blob/main/workspace/memory/architecture.md))

- **REST for the public surface.** JSON over HTTPS, OpenAPI 3.1.
- **Versioned in the URL** (`/v1/`, `/v2/`) for breaking changes. Additive changes are unversioned.
- **Idempotency keys** on every mutating call. A retry must be a no-op.
- **Errors are typed**, not strings: `{ "error": { "code": "TENANT_NOT_FOUND", "message": "...", "request_id": "..." } }`. Codes are stable; messages are not.

## The base URL

```
https://<your-forge-host>/api/v1
```

In dev: `http://localhost:3000/api/v1`.

## Authentication

The API uses **Bearer tokens**. The token is a per-tenant, time-bounded JWT (≤ 15 min).

```bash
curl -H "Authorization: Bearer $Forge AI_API_TOKEN" \
     -H "X-Idempotency-Key: $(uuidgen)" \
     https://fora.example.com/api/v1/runs
```

Get a token from the Forge console → Settings → API tokens. A token is scoped to:

- A tenant (`tenant_id`)
- A set of permissions (`runs:read`, `runs:write`, `audit:read`, `tenants:admin`)
- A TTL (≤ 15 min)

## Top-level endpoints

| Endpoint | Method | Description |
| --- | --- | --- |
| `/healthz` | `GET` | Liveness check |
| `/readyz` | `GET` | Readiness check |
| `/api/v1/runs` | `GET` | List runs (filter: `tenant_id`, `stage`, `status`) |
| `/api/v1/runs` | `POST` | Create a new run |

## Idempotency

Every mutating call takes an `X-Idempotency-Key` header. The orchestrator stores `(tenant_id, idempotency_key) → response` for 24 h. A retry with the same key returns the cached response.

```bash
# First call
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "X-Idempotency-Key: 01HXYZ..." \
     -d '{"type":"feature","prompt":"Add /health endpoint"}' \
     https://fora.example.com/api/v1/runs

# Retry (network blip) — same key, same response
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "X-Idempotency-Key: 01HXYZ..." \
     -d '{"type":"feature","prompt":"Add /health endpoint"}' \
     https://fora.example.com/api/v1/runs
```

## Errors

```json
{
  "error": {
    "code": "TENANT_NOT_FOUND",
    "message": "No tenant with id 'acme-corp' was found in your account.",
    "request_id": "01HXYZ...",
    "documentation_url": "https://docs.fora.ai/api/errors/TENANT_NOT_FOUND"
  }
}
```

| Code | HTTP | Meaning |
| --- | --- | --- |
| `UNAUTHENTICATED` | 401 | Missing or invalid token |
| `FORBIDDEN` | 403 | Token does not have the required scope |
| `NOT_FOUND` | 404 | Resource does not exist |
| `TENANT_NOT_FOUND` | 404 | Tenant slug does not exist |
| `IDEMPOTENCY_KEY_REUSED` | 409 | The same key was used with a different payload |
| `COST_CEILING_EXCEEDED` | 422 | The run would exceed the per-run cost ceiling |
| `GATE_REJECTED` | 422 | The previous stage's gate was rejected |
| `RATE_LIMITED` | 429 | Too many requests |
| `INTERNAL` | 500 | Server error; check `request_id` |

## Rate limits

- Default: 100 RPS per token
- Configurable per-token via `RATE_LIMIT_RPS` env
- Headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`

## Pagination

List endpoints are cursor-paginated:

```bash
GET /api/v1/runs?limit=50&cursor=eyJ0IjoxNzE4NDAwMDAwfQ
```

The response includes `next_cursor` and `prev_cursor`.
