## What this ships

`agents/workspace/version.py` — a deterministic, dependency-free
version train CLI that tags every change to the Knowledge Layer seed,
records a monotonic tag history, and restores the seed to any prior
tag without ever touching the tenant's overrides.

The implementation is the smallest credible shape for the spec:

- A single manifest at `workspace/workspace-manifest.json`
  (`schemaVersion: 1`, `currentTag`, monotonic `tags[]`, per-file
  SHA-256 manifest).
- Plain-file snapshots under `workspace/.tags/<name>/<relpath>` — no
  opaque blobs, git diff-able out of the box.
- Five subcommands: `init`, `tag`, `rollback`, `list`, `status`.
- A non-skippable smoke gate wired into `scripts/smoke.sh` after the
  Forge AI-408 lint gate.

## Why plain-file snapshots

A spec that says "git diff-able history (no opaque blobs)" rules out
hash-only records and base64 blob storage. Plain files under
`.tags/<name>/<relpath>` make every tagged version diffable with
`git diff v0..v1` (or, in practice, with `diff -ru .tags/v0 .tags/v1`)
and keep the manifest itself small (just hashes + size, not content).

## CLI

```text
python -m agents.workspace.version tag --root workspace/ --message "..."
python -m agents.workspace.version rollback --root workspace/ --to v1
python -m agents.workspace.version list --root workspace/
python -m agents.workspace.version status --root workspace/ [--json]
python -m agents.workspace.version init --root workspace/
```

Exit codes: `0` success, `1` logic error (unknown tag, duplicate name,
post-rollback hash mismatch, protected-path violation), `2` usage error.

## Critical invariant — tenant overrides survive rollback

`workspace:rollback` ONLY operates on the seed. It never reads,
writes, or copies into `tenants/<slug>/workspace/...` or
`engagements/<slug>/...`. The check is enforced at two layers:

1. The rollback loop iterates the tag's file manifest only — there is
   no "walk the whole tree" step that could cross into a tenant path.
2. `_is_protected(relpath)` rejects any relpath that starts with
   `../`, `tenants/`, `engagements/`, or contains `/tenants/` or
   `/engagements/`. Both `capture_tag` and `rollback_to` call it as a
   defense-in-depth check.

The smoke test (AC3) creates a tenant override file SIBLING to the
seed root (`tenants/acme/workspace/customer/standards.md`), rolls
back, and asserts SHA-256 byte-identical content.

## What this does NOT do (deferred)

- **Cross-tag diff UX** — `git diff -ru .tags/v0 .tags/v1` is the diff
  tool today. A `workspace:diff --from v0 --to v1` wrapper is a
  v0.1.1 convenience, not a v0.1 blocker.
- **Pack / prune** — old tags stay on disk forever; there is no `gc`.
  Acceptable for the first version: tags are tiny (plain files in the
  seed) and a tag is itself a recoverable artifact.
- **Multi-workspace federation** — the manifest is per-`workspace/`,
  not multi-tenant. Forge AI-103's "per-tenant materialisation" is the
  layer that wraps this.

## Risk + rollback

Risk: low. The version train operates on the seed only; the manifest
itself is inside the seed and is itself included in subsequent tags
via the snapshot copy. A bad rollback is recoverable by tagging the
pre-rollback state as vN+1 and rolling back again.

To revert this change in an emergency: delete
`agents/workspace/version.py` and
`agents/workspace/smoke_test_version.py` and revert the `scripts/smoke.sh`
hunk. No data on customer systems is touched.
