<objective>
Redesign the 8 center pages using the new design tokens, semantic shell primitives, and curated Phase 0.5 amendment styling. Each page must:

- Use a shared `PageHeader` primitive (eyebrow + title + description + optional primary action) instead of bespoke header markup
- Use semantic tokens (`text-foreground`, `text-muted-foreground`, `bg-card`, `text-subtle`) — no `forge-50/100/200/300/...` literal classes for new content
- Replace any `.card` utility usages (legacy) with `<SectionCard>` wrapping shadcn `Card` primitives OR explicitly pass semantic classes
- Replace ad-hoc status chips with `StatusPill` (or its wrappers — `RunStatusBadge`, `HealthBadge`, `ApprovalStatusBadge`, etc.) — never color alone
- Use `<EmptyState>` for empty data states (empty connections, no audits, no agents, etc.)
- Keep all existing `data-testid` attributes intact so tests in `apps/forge/tests/` continue to pass

This plan does NOT introduce new shadcn primitives, does NOT build a `<Sidebar>` (that's 0.5-03's job), and does NOT introduce a new layout shell — it operates inside the existing per-page `AdminShell` pass-through wrapper which already yields the persistent sidebar from `app/layout.tsx`.

Source spec: curated design vision (Phase 0.5 amendment, 2026-06-23) §6 Agent Status Language; tokens in `apps/forge/lib/design-system/forge-color-tokens.ts`; semantic mapping in `apps/forge/lib/design-system/status.ts`; Tailwind binding in `apps/forge/tailwind.config.ts`; CSS variables in `apps/forge/app/globals.css`.

</objective>
