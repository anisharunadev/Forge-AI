<objective>
Land the data primitives the 8 center pages need to surface real pilot data: a typed DataTable (TanStack Table v8) with sorting/filtering/pagination/row-selection, a typed DataToolbar (search + filter + sort + bulk actions), a typed Form primitive on top of react-hook-form + zod wired through shadcn form.tsx, theme-aware Recharts wrappers (LineChartCard, BarChartCard, AreaChartCard, PieChartCard) that read from semantic design tokens, and a shared EmptyState primitive. Add the missing shadcn primitives (table, pagination, breadcrumb, avatar, accordion, form, radio-group, switch, slider) so all later phases can compose against a stable surface.

This plan does NOT redesign any center page or wire TanStack Query. It establishes the data primitives that Plans 0.5-05/06/07 compose against.

Source spec: curated design vision (Phase 0.5 amendment, 2026-06-23) section 4 Data Surfaces — every data-bearing surface must be visual, sortable, filterable, paginated, and have an empty state.
