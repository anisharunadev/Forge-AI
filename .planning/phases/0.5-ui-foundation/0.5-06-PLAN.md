<objective>
Build the AI-native + visualization surfaces that distinguish Forge from a CRUD app: typed React Flow node components with status-based coloring, four graph views (Knowledge, Repository, Workflow, Agent Execution), a virtualized Audit Timeline that handles >1000 events smoothly, an Approval Timeline with one-click approve/reject, and three AI-native panels (TokenStream, ToolCallCard, AgentTraceTimeline) for live agent execution.

This plan does NOT touch the existing bespoke hex-color React Flow views in `components/knowledge-center/`, `components/architecture/`, `components/ideation/`, or `components/project-intelligence/`. Those keep working. The new graph primitives live in `components/graph/` and `components/ai-native/`; they use the typed node pattern (`nodeTypes={{ artifact: ArtifactNode }}`) so the page-level views stay declarative. Wiring happens in (j): swap `KnowledgeGraphView` in `app/knowledge-center/page.tsx` with the new typed version, and add the virtualized AuditTimeline to `app/audit/page.tsx`.

Source spec: curated design vision (Phase 0.5 amendment, 2026-06-23) §6 Agent Status Language — color + glyph + pulse + label, never color alone.

</objective>
