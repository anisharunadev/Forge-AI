<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sub-graph caller → external static-analysis tool | The sub-graph invokes ruff/mypy/bandit via subprocess. The tool output is parsed and trusted as evidence. |
| Sub-graph → audit_service | Every node writes one audit row. A missed write = unrecoverable gap. |
| Code Validator → Approval gate | The sub-graph runs INSIDE the IMPLEMENTATION phase and is gated by `@require_approval_phase`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-1 | Tampering | Subgraph tool output | mitigate | Each node parses tool JSON output via Pydantic models (LintFinding, etc.); malformed entries are dropped with an audit row recording the parse failure. |
| T-01-05-2 | Denial of Service | Tool subprocess | mitigate | Each subprocess call has a timeout (60s for ruff, 120s for mypy/bandit). On timeout, the node writes an empty findings list + an audit row with `result={"tool_failed": True, "reason": "timeout"}`. |
| T-01-05-3 | Elevation of Privilege | LLM call from code_validator | mitigate | The sub-graph is deterministic — no `litellm_client.completion` call exists in `agents/code_validator/**`. Test `test_subgraph_no_llm_call` proves this with a mock. |
| T-01-05-4 | Repudiation | Verdict computation | mitigate | Verdict is computed by the `emit_report` node from the union of finding lists; the rule is deterministic and documented in code. |
| T-01-05-5 | Tampering | Code Validator entry bypass | mitigate | Entry function carries `@require_approval_phase(SDLCPhase.IMPLEMENTATION)` (from plan 01-01). A direct call without approval raises `ApprovalRequiredError`. |
| T-01-05-SC | Tampering | npm/pip/cargo installs | mitigate | ruff/mypy/bandit must be verified as already installed in dev env. If absent, add a `<task type="checkpoint:human-verify">` to confirm. The plan adds no new third-party deps beyond standard tooling. |
</threat_model>
