Wire F-503 (Merge Gate — rules-only, LLM excluded per the locked Phase 1 decision in STATE.md) and F-505 (Tool Bundles — typed allowed-tool set per agent). Output: A rules-only `MergeGateEngine` that emits `MergeGateDecision(verdict, evidence, blockers)`, and a `ToolBundlesRegistry` that maps `agent_name → list[tool_name]` and gates every MCP call.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Merge Gate → Code Validator | The gate trusts the ValidationReport verdict. A bad report = bad gate decision. |
| MCP router → ToolBundlesRegistry | The router must consult the registry BEFORE invoking any tool. A bypass = privilege escalation. |
| Agent → tool bundle | The agent's identity is asserted by the JWT principal; the bundle lookup must use that identity, not a query param. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-1 | Elevation of Privilege | Merge Gate LLM call | mitigate | `MergeGateEngine` is rules-only — no `litellm_client.completion` call exists in the file. Test `test_no_llm_call` proves it. |
| T-01-06-2 | Tampering | ValidationReport spoofing | mitigate | The Merge Gate calls the Code Validator directly (in-process); the `validation_report_id` is the UUID returned by the validator, not user input. |
| T-01-06-3 | Elevation of Privilege | Tool bundle bypass | mitigate | The MCP router's `enforceToolBundle` runs BEFORE dispatch. Default `default_allow=False` means unknown agents get no tools. |
| T-01-06-4 | Repudiation | Merge gate decision | mitigate | `MergeGateDecision` is a typed artifact; the audit row carries `agent="merge_gate"`, `tool="code_validator"`, `artifact={"validation_report_id": ...}`. |
| T-01-06-5 | Spoofing | Agent identity | mitigate | The MCP router reads agent identity from the JWT principal (not query string). The bundle lookup is `agent_name → principal.name`. |
| T-01-06-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] packages; this plan adds no new third-party Python deps. |
</threat_model>
