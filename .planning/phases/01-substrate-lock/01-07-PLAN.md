<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Approval gate → WorkflowBudget | The gate reads budget snapshot via `surface_at_gate`. A stale snapshot = wrong decision. |
| Bootstrap → connector_manager | Bootstrap trusts the connector handshake result. A spoofed handshake = false `ready_for_pilot`. |
| Bootstrap → LLM provider | Bootstrap calls `litellm_client.completion` with a 1-token ping. A failure = honest `handshake_ok=False`. |
| Bootstrap → knowledge_graph | Bootstrap writes seeded nodes via `knowledge_graph.upsert_node`. A direct SQL bypass = loss of typed-artifact guarantee. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07-1 | Repudiation | WorkflowBudgetSnapshot | mitigate | The snapshot is a typed Pydantic model with `snapshot_at` timestamp. Audit row records the snapshot's UUID + tenant_id. |
| T-01-07-2 | Tampering | Budget adjustment | mitigate | `adjust_budget` is decorated with `@require_approval_phase(SDLCPhase.PLANNING)`. A direct call without approval raises `ApprovalRequiredError`. |
| T-01-07-3 | Spoofing | Bootstrap handshake | mitigate | Both connector and LLM handshake results come from typed managers, not user input. `handshake_ok=False` → `ready_for_pilot=False`. |
| T-01-07-4 | Tampering | KG node creation | mitigate | Bootstrap goes through `knowledge_graph.upsert_node(...)` (no direct SQL). The KG layer enforces tenant_id + project_id scoping (Rule 2 + Rule 5). |
| T-01-07-5 | Information Disclosure | Per-tenant budget cap | accept | The dict lives in `Settings` (in-memory only); not exposed via API. |
| T-01-07-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] packages; this plan adds no new third-party Python deps. |
</threat_model>
