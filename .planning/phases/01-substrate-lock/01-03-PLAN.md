<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Phase node → audit_log | Every mutation must flow through `audit_service.record`. A skipped write = unrecoverable audit gap. |
| Backend → OTel collector | OTLP exporter must be reachable; if misconfigured, traces silently drop. |
| Health probe caller → /healthz | Operators rely on `/healthz` to detect degraded observability before cutover. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-1 | Repudiation | BasePhaseNode.mutate | mitigate | `mutate` calls `audit_service.record` unconditionally — no `if` guard, no opt-out flag. PITFALL-5 closure. |
| T-01-03-2 | Tampering | audit_log table | mitigate | `record()` signature is keyword-only and required; missing any field raises `TypeError` at call site, not silently dropped. |
| T-01-03-3 | Information Disclosure | OTLP exporter endpoint | accept | The endpoint is internal infrastructure (otel-collector sidecar). Not exposed to untrusted networks in dev/staging/prod topologies. |
| T-01-03-4 | Denial of Service | /healthz probe | accept | The handler is in-memory only (no DB calls). Constant-time response. |
| T-01-03-5 | Tampering | Health probe downgrade attack | mitigate | Production env requires `audit_sink != "disabled"` AND `otel_exporter_configured == True` else HTTP 503. Operators cannot accidentally cutover a degraded substrate. |
| T-01-03-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] packages; this plan adds no new third-party Python deps. |
</threat_model>
