Wire F-601 (Refactor Agent) to emit a typed `MigrationPlan` artifact consumed by the Code Validator's diff baseline, and F-504 (Steering Rules Engine) to evaluate steering policies against an SDLC state and emit a `SteeringDecision`. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Refactor Agent → tool_bundles | Every tool call must consult the bundle. A bypass = privilege escalation. |
| Steering Rules → knowledge_graph | The engine queries the KG for evidence. A malicious policy could exfiltrate data; mitigations: policy YAML is curated, queries are read-only. |
| Refactor Agent → LLM | The agent invokes LLM for refactor proposals. Rule 1: must go through `litellm_client`. Rule 6: every call writes an audit row. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08-1 | Elevation of Privilege | Tool bundle bypass | mitigate | Both Refactor Agent and Steering Rules Engine call `tool_bundles.is_tool_allowed(...)` before any tool invocation. Out-of-bundle raises `ToolBundleViolationError`. |
| T-01-08-2 | Tampering | LLM response | mitigate | The LLM response is parsed into `FileChange` records via a Pydantic validator; malformed entries are rejected. Risk is computed deterministically from the parsed records. |
| T-01-08-3 | Information Disclosure | Steering policy | accept | Policy YAMLs are curated by the architecture team; not user-editable in pilot. |
| T-01-08-4 | Repudiation | MigrationPlan / SteeringDecision | mitigate | Both are typed Pydantic v2 artifacts with `produced_at`/`decided_at` timestamps and `to_kg_payload()` for React Flow rendering. Audit row carries the full artifact metadata. |
| T-01-08-5 | Spoofing | Refactor Agent entry | mitigate | `propose_refactor` is decorated with `@require_approval_phase(SDLCPhase.IMPLEMENTATION)` (from plan 01-01). A direct call without approval raises `ApprovalRequiredError`. |
| T-01-08-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] packages; this plan adds no new third-party Python deps. |
</threat_model>
