<objective>
Wire pre-call cost admission into `litellm_client.py`: derive per-call projected cost from prompt-token estimate × model price, enforce a cumulative cap, write a row to `cost_ledger` (per ADR-009), and surface "Run budget: $X / Used: $Y" in the UI before a run starts (closing PITFALL-2).

Purpose: PITFALL-2 is the silent-budget-exceedance gap — today a misbehaving agent can drive a run's LLM cost past the ceiling with no advance warning and no hard stop. The substrate must project cost before every LLM call, reject the call if it would exceed the cap, and show the budget state to the operator before they click "Start run".

Output: `litellm_client.py` gains a `pre_call_admission(...)` helper that reads prompt-token estimate + model price, looks up the cumulative ledger sum, and rejects if projected > remaining. A `RunBudgetBadge` React component shows ceiling/spent/remaining before run start.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM caller → litellm_proxy | All LLM traffic must pass through pre_call_admission before hitting LiteLLM. |
| Substrate → cost_ledger table | Append-only; every projected row is paired with an actual row. |
| Operator UI → budget endpoint | Reads ceiling + spent; does not write. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-1 | Denial of Service | Cost-cap bypass | mitigate | `pre_call_admission` runs before every LLM call. The existing `completion()` function calls admission first; if denied, raises `CostCapExceeded` and never hits LiteLLM. |
| T-01-02-2 | Tampering | Cost ledger rows | mitigate | `record_projected` / `record_actual` use keyword-only required args (Rule 2); no `= None` defaults. Missing fields raise `TypeError`. |
| T-01-02-3 | Information Disclosure | Per-tenant budget overrides | accept | The dict lives in `Settings` (in-memory only); not exposed via API. |
| T-01-02-4 | Repudiation | Projected vs actual cost | mitigate | PITFALL-2 closure: every LLM call writes a projected row first; the actual cost row is written post-call. The delta is auditable. |
| T-01-02-5 | Tampering | Budget GET endpoint | mitigate | The endpoint is decorated with `@require_approval_phase(SDLCPhase.PLANNING)` (from plan 01-01). Tenant scoping is read from the JWT principal, not the query string. |
| T-01-02-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] packages; pyyaml must be verified as already in `requirements.txt`. If absent, add a checkpoint before install. |
</threat_model>
