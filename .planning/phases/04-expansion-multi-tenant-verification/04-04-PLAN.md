## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cursor client to Forge Backend | Anonymous client (no JWT); Forge must identify tenant via X-Forge-Tenant header |
| Forge Backend to LiteLLM | Forge injects admin key + metadata envelope; LiteLLM attributes spend per forge_tenant_id |
| Admin user to provider enable | Privilege escalation if non-admin can toggle providers |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-04-04-01 | Information Disclosure | spend misattribution (admin key bypasses Virtual Key) | high | mitigate | metadata.forge_tenant_id injected into body so LiteLLM reconciles against it |
| T-04-04-02 | Elevation of Privilege | non-admin enables provider for tenant | medium | mitigate | PATCH /providers/{name} requires require_permission("forge.providers.admin") |
