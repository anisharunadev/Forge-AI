<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Forge Backend to LiteLLM /routes | Internal service-to-service; requires master key in Authorization header |
| Admin user to /providers/refresh | Privilege escalation if non-admin can force refresh; mitigated by require_permission("forge.providers.admin") |
| Admin UI to GET /api/forge/providers | Read-only; uses user's existing auth context; no extra privilege |
| Redis cache to admin user | Cache key is global (not per-tenant) — the catalog is platform-level. If multi-tenant isolation of the catalog is ever required, key must become forge:routes:catalog:{tenant_id} |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-18-01 | Denial of Service | /routes called on every GET /api/forge/providers | high | mitigate | Redis cache with 5s TTL; two consecutive GETs within 5s result in a single /routes call (test_redis_cache_hit_within_5s proves it) |
| T-04-18-02 | Elevation of Privilege | non-admin force-refreshes catalog to overwrite other tenants' state | high | mitigate | require_permission("forge.providers.admin") on POST /refresh; cache key is global, but the operation is admin-gated; audit CATALOG_REFRESHED with actor_id |
| T-04-18-03 | Information Disclosure | route catalog leaks internal LiteLLM upstream paths to non-admin UI | low | accept | The route catalog is intentionally exposed to admins (it powers the ProvidersTab); the UI gating is admin-only via the existing /admin/settings nav |
| T-04-18-04 | Repudiation | /routes call not attributed to tenant | medium | mitigate | metadata envelope (forge_tenant_id, forge_user_id, forge_run_id, kind: 'providers.catalog.fetch') injected on the /routes call; CATALOG_REFRESHED audit row carries actor_id and tenant_id |
| T-04-18-05 | Denial of Service | /providers/refresh hit in tight loop by misbehaving admin | low | mitigate | Phase 3 RBAC already rate-limits admin operations; documented in D-5 |
| T-04-18-06 | Tampering | admin lowers FORGE_ROUTE_CATALOG_TTL_SECONDS to 0 to defeat cache | low | accept | Pydantic validator rejects ttl < 1; cache miss is acceptable behavior; the rate-limit is the slow-loop defense |

</threat_model>
