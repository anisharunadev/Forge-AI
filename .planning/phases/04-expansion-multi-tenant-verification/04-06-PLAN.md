(2) POST /audio/transcriptions → reads UploadFile from Form, sends multipart to LiteLLM /v1/audio/transcriptions; returns JSON. (3) POST /audio/speech → POST /v1/audio/speech, returns Response with audio/mpeg content. (4) POST /videos → POST /v1/videos, returns {job_id, status}. (6) POST /moderations → POST /v1/moderations, parses response.json(), records audit row with category_scores in payload, returns JSON. Each route calls audit_service.record with the corresponding Phase4AuditAction constant (MEDIA_IMAGE_GENERATED, MEDIA_AUDIO_GENERATED, MEDIA_AUDIO_TRANSCRIBED, MEDIA_VIDEO_QUEUED, MEDIA_MODERATION_COMPLETED). If MEDIA_AUDIO_TRANSCRIBED does not exist, add it. In backend/tests/phase4/test_media_router.py, write one test per endpoint with mocked httpx.
