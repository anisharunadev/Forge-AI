<objective>
Close F19 AC4 ("Semantic cache returns the cached response for queries with similarity >= 0.95"). Wraps the new cache_type='semantic' (and 'auto' picker) into chat_complete_cached. Cross-tenant isolation re-verified; PII bypass re-applied at the embedding call. Audit row forge.cache.hit extended with hit_type='semantic' + similarity_score.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM call to LiteLLM /embeddings | Forge injects metadata envelope (forge_tenant_id, forge_user_id, forge_run_id) so spend + audit attribution works for embedding calls too (Rule 6 + Research Pitfall 7) |
| Tenant A semantic cache to Tenant B probe | Phase4CacheKey SELECT always filters on tenant_id from the principal; even if embedding distance is < 0.05 across tenants, the query cannot return the row |
| Admin to /refresh-semantic endpoint | Mass-embed abuse; require_permission("forge.cache.admin") + tenant_id in body must match principal's tenant |
