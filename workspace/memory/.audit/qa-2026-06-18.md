## 3. Anti-pattern surface

`grep -niE "tbd|todo|fixme|it depends|placeholder|tribal"` over `qa.md` returns **zero matches**.

The §3 disclosure of the still-`proposed` ADR-0004 status is the correct *opposite* of the anti-pattern: the file calls out the gap and the consumer (cold-started sub-agents go to §3, not the missing ADR file). No "we'll get to it" or "it depends" language.

Spot-check of the qualitative §3 acceptance-bar criteria:

- **"A sub-agent, woken cold with only that file in context, can read it and act."** — yes; §1 names the gates above and below, §2 names every test tier, §3 names the v1 contract, §5 names the handoff to Security, §7 names the smoke test command, §8 names every issue and owner.
- **"No tribal knowledge."** — yes; the §3 ADR-0004 status is *disclosed*, not assumed; the §6 Phase 4 self-healing scope is gated on the `RepairProposal.validate()` guardrail, not on a future conversation.
- **"Opinionated. 'It depends' is not in a v1 file."** — yes; every rule is unambiguous ("the four tiers are mandatory", "downgrading is a CTO call, not an agent call", "a `not_implemented` tier on a critical path is a Security-block, not a QA-pass", "do not retry on a 4xx").
- **"`Related` section at the bottom."** — yes; lines 133-142.
